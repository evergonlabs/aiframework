Overview

What aiframework does

Your repo already contains everything an AI assistant needs to be effective — it just can't read it yet. aiframework bridges that gap by scanning your codebase and generating a complete set of AI configurations that work with Claude Code, Cursor, Codex, and Copilot.

The framework operates across four layers, each handling a different part of the AI development lifecycle:

Scanning & Analysis

13 scanners read your language, framework, commands, directory structure, CI/CD, environment variables, code quality tools, domain concerns, code symbols, project archetype, and more. The output is a deterministic manifest — same repo, same result, every time.

Config Generation

From the manifest, aiframework generates CLAUDE.md (your project brain), .claude/rules/ (auto-loaded workflow rules), .claude/skills/ (slash commands), git hooks (auto-lint, auto-refresh), CI workflows, documentation scaffolds, and cross-tool configs for Cursor and Copilot.

Runtime Learning

As you work, the framework captures learnings from your coding sessions. A bidirectional bridge syncs insights between formats. Weekly evolution sessions analyze patterns and propose improvements to your CLAUDE.md rules. Your AI assistant gets smarter over time.

Persistent Memory

A vault knowledge base persists decisions, wiki pages, session retros, and operational context across every session. Claude Code reads it at the start of each session, so context is never lost between restarts.

Process

How it works

The framework follows a five-phase improvement loop. Each phase feeds into the next, creating a continuous cycle where your AI configurations become more precise with every coding session.

1

Bootstrap

aiframework run --target ~/your-project

Scans your entire repository with 13 specialized scanners. Detects language, framework, package manager, test commands, CI provider, environment variables, domain concerns (auth, database, API, AI/LLM, etc.), and project archetype. Produces a deterministic manifest and generates all configuration files in about 60 seconds.

2

Work

/aif-review /ship /qa /aif-learn

You code with Claude Code using 52+ slash commands. Review code against your project's invariants, ship with automated lint-test-changelog-commit workflows, test in a real headless browser, and capture learnings as you discover gotchas. Learnings are stored in both JSONL format and synced to runtime intelligence tools.

3

Session Retrospective

/sheal-retro

At the end of each coding session, the runtime intelligence layer analyzes what happened — which patterns repeated, what failed, what worked. A bidirectional bridge converts these insights between formats so they are available to all parts of the framework. Retro insights are synced to the persistent vault.

4

Evolution

/aif-evolve

Weekly, the evolution skill reads all accumulated learnings from both the JSONL store and the runtime intelligence layer. It cross-references patterns, identifies repeated friction, and proposes concrete improvements to your CLAUDE.md — new invariants, updated rules, cleanup of stale entries. Changes are applied only with your approval.

5

Drift Detection

/sheal-drift

Detects when learnings are captured but not being followed. If the same mistake recurs three or more times, the learning is promoted from a temporary note to a permanent CLAUDE.md invariant or a .claude/rules/ file. The pre-push hook auto-refreshes configs when source files change. The loop closes and repeats.

Capabilities

What you can do

52+ slash commands organized by workflow phase. Every command works out of the box after running /aif-ready.

Setup & Config

/aif-ready researches your stack, enhances CLAUDE.md, and optimizes settings. /aif-enhance fills gaps. /aif-pulse discovers new Claude Code features monthly.

Ship & Review

/ship runs tests, reviews diff, bumps version, creates PR. /review checks for SQL safety, injection risks, and invariant violations. /land-and-deploy merges and verifies production.

QA & Browser

/qa finds and fixes bugs using a real headless browser. /browse navigates pages at ~100ms per command. /benchmark tracks Core Web Vitals per PR. /canary monitors post-deploy.

Debug & Investigate

/investigate runs 4-phase root cause debugging. /health gives a 0-10 code quality score. /cso runs OWASP + STRIDE security audits. /retro provides weekly engineering retrospectives.

Plan & Design

/plan-ceo-review for founder-mode scope review. /plan-eng-review locks architecture. /design-shotgun generates design variants. /design-html produces production HTML/CSS.

Learn & Evolve

/aif-learn captures gotchas. /aif-evolve proposes rule improvements. /sheal-drift detects unapplied learnings. /sheal-retro extracts session patterns. The framework improves itself.

Dependencies

What powers the framework

aiframework integrates with three complementary tools. Only the core framework is required — everything else enhances it and installs with a single command.

Component Purpose Installs via Status
aiframework Core scanner, generator, and verification engine. Produces CLAUDE.md, rules, skills, hooks, vault, and CI configs from any repo. make install Required
gstack 37 additional skills for shipping, QA, browser testing, design, planning, and security. Adds a persistent headless Chromium daemon. git clone + ./setup Recommended
sheal Runtime session intelligence. Watches AI coding sessions, extracts learnings, detects failure patterns, and bridges insights back into aiframework. Auto with make install Optional

Getting Started

Install in three steps

From zero to a fully AI-ready repo in about two minutes.

Install the framework

This installs aiframework and automatically attempts to install sheal (optional, requires Node.js). If Node.js is not available, sheal is skipped and everything else works normally.

git clone https://github.com/evergonlabs/aiframework.git && cd aiframework && make install
git clone --depth 1 https://github.com/garrytan/gstack.git ~/.claude/skills/gstack && cd ~/.claude/skills/gstack && ./setup --no-prefix

Bootstrap your project

Point aiframework at any repo. It scans everything, generates all configs, and verifies the result. This is deterministic — same repo produces the same output every time.

aiframework run --target ~/your-project

Start coding with Claude Code

Open your project in Claude Code and run the one-time enhancement command. After this, Claude knows your stack, your commands, your invariants, and your architecture.

cd ~/your-project && claude

# Then in Claude Code:
/aif-ready

After setup, your CLAUDE.md auto-refreshes on every git push. Run /aif-evolve weekly to improve rules from accumulated learnings. Run /aif-pulse monthly to discover new Claude Code features.
