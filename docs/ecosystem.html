🏗 aiframework — Your Repo's Brain Builder

Run once per project. aiframework reads your repo the way a senior engineer would on their first day — except it takes 60 seconds instead of a week.

  • 13 scanners detect language, framework, CI, domains, architecture, and more
  • Generates CLAUDE.md (80-150 lines of high-signal project context)
  • Creates .claude/rules/ with workflow, testing, and security rules
  • Builds custom /review and /ship skills tailored to your stack
  • Sets up git hooks that auto-refresh configs on every push
  • Creates a 31-file vault knowledge base for persistent memory
  • Works with ANY language: TypeScript, Python, Go, Rust, Ruby, Java, and 14 more

🌐 gstack — Your Engineering Team in 37 Skills

Gives Claude Code superpowers: ship code, test in a real browser, review PRs, plan features, audit security, and run retrospectives.

  • /ship — test, review, bump version, create PR in one command
  • /qa — find bugs in a real browser, fix them, verify the fix
  • /browse — headless Chromium at ~100ms per command
  • /investigate — 4-phase root cause debugging
  • /plan-ceo-review — founder-mode scope expansion
  • /plan-eng-review — lock in architecture before coding
  • /design-review — visual QA with auto-fix
  • /cso — OWASP + STRIDE security audit

🔄 sheal — Runtime Session Intelligence

Learns from your actual coding sessions. Detects patterns, extracts failures, and bridges learnings back into aiframework so your CLAUDE.md gets smarter over time.

  • /sheal-check — health check at session start (tests, deps, environment)
  • /sheal-retro — extract learnings from the session, bridge to JSONL
  • /sheal-drift — detect learnings that keep being ignored, promote to rules
  • /sheal-ask — search past session history with natural language
  • Bidirectional bridge syncs learnings between JSONL and markdown formats
  • Category mapping: bug/gotcha/pattern/decision preserved losslessly
  • Optional — zero behavioral change when not installed

🧠 Agentic Memory (Vault) — Persistent Knowledge

Claude Code loses memory between sessions. The vault fixes that. Decisions, wiki pages, retro insights, and operational notes persist forever.

  • vault/memory/status.md — operational context, checked at every session start
  • vault/memory/decisions/ — architectural decision records (ADRs)
  • vault/memory/notes/ — persistent notes across sessions
  • vault/wiki/ — auto-generated knowledge pages from code index
  • vault/templates/ — reusable templates for decisions and source pages
  • vault-tools.sh doctor — full vault health diagnostic
  • Retro insights from sheal auto-sync to vault/memory/status.md

How They Work Together

A complete improvement loop. Your AI coding assistant gets smarter with every session — automatically.

1

Bootstrap

aiframework

You run aiframework run once. It scans your entire repo — language, framework, commands, CI, domains, architecture — and generates everything Claude Code needs: CLAUDE.md, rules, skills, hooks, vault, and CI workflows.

2

Work

gstack aiframework skills

You code with Claude Code. Use /ship to deploy, /review to check code, /qa to test in a real browser. When you discover something, /aif-learn captures it to both JSONL and sheal simultaneously.

3

Retro

sheal

At session end, /sheal-retro analyzes what happened. It extracts failure patterns, recurring mistakes, and insights. The bidirectional bridge syncs everything to your JSONL learnings store and the vault.

4

Evolve

aiframework sheal

Weekly, /aif-evolve reads all learnings from both sources. It proposes CLAUDE.md improvements: new invariants, new rules, cleanup of stale entries. Your project brain gets smarter.

5

Drift Detection

sheal vault

/sheal-drift checks: are learnings being applied? If the same mistake keeps happening (3+ times), it gets promoted to a permanent CLAUDE.md invariant. Retro insights sync to vault memory. The loop closes.

Get Started in 3 Steps

Two minutes from zero to a fully AI-ready repo.

Step 1: Install everything

# aiframework + sheal (auto-installed if npm available)
git clone https://github.com/evergonlabs/aiframework.git
cd aiframework && make install

# gstack (37 skills for shipping, QA, design)
git clone --depth 1 https://github.com/garrytan/gstack.git ~/.claude/skills/gstack
cd ~/.claude/skills/gstack && ./setup --no-prefix

Step 2: Bootstrap your project

aiframework run --target ~/your-project

This scans your repo with 13 scanners, generates CLAUDE.md + 23 files, and verifies everything is correct. Takes about 60 seconds.

Step 3: Open Claude Code

cd ~/your-project && claude
# Then type:
/aif-ready

That's it. Claude now knows your stack, commands, invariants, and architecture. Start coding.
