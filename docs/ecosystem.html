How It Works

Four layers, one framework

Each layer handles a different phase of the AI development lifecycle. Together, they create a feedback loop that makes your AI assistant smarter with every session.

🔍

Scanning & Analysis

13 scanners read your language, framework, commands, CI, environment, domains, and architecture. The output is a deterministic manifest — same repo, same result, every time.

⚙

Config Generation

Generates CLAUDE.md, .claude/rules/, .claude/skills/, git hooks, CI workflows, documentation scaffolds, and cross-tool configs for Cursor and Copilot.

🔄

Runtime Learning

Captures learnings from coding sessions. A bidirectional bridge syncs insights between formats. Weekly evolution proposes improvements to your rules.

🧠

Persistent Memory

A vault stores decisions, wiki pages, retro insights, and operational context across every session. Context is never lost between restarts.

Automation

What happens automatically

After setup, you just code. The framework handles health checks, type checking, linting, quality gates, config refresh, and session retros — all via hooks.

1

Session Opens

Automatic

SessionStart hook fires. Runs a health check (sheal check or aiframework verify). You see project status immediately — no command needed.

2

You Edit Code

Automatic

PostToolUse hook fires on every file edit. Runs type checking for TypeScript, Rust, Go, or Python projects. Errors surface in real-time.

3

You Commit & Push

Automatic

pre-commit hook runs lint. pre-push hook runs quality gate (lint + test + build), config drift detection, and auto-refresh.

4

Session Closes

Automatic

Stop hook fires. Runs session retrospective via sheal (if installed) to extract learnings. Or surfaces a reminder to capture learnings manually.

5

Weekly Evolution

Manual — by design

/aif-evolve reads all learnings and proposes CLAUDE.md improvements. Requires your approval before changing rules. This is a safety gate, not a missing feature.

Dependencies

What powers the framework

One required install. Everything else is optional and enhances the experience.

ComponentPurposeInstallStatus
aiframework Core scanner, generator, and verification engine make install Required
gstack 37 skills for shipping, QA, browser testing, design, and security git clone + ./setup Recommended
sheal Runtime session intelligence — watches sessions, extracts learnings Auto with make install Optional
Vault Persistent memory — decisions, wiki, retros across sessions Auto-generated Automatic

System requirements: bash 3.2+ · jq 1.6+ · git 2.0+ · Python 3.10+ (recommended) · Node.js 18+ (only for sheal)

Getting Started

Three steps to AI-ready

From zero to a fully configured repo in about two minutes.

1. Install the framework

Installs aiframework. Automatically attempts to install sheal and gstack.

git clone https://github.com/evergonlabs/aiframework.git && cd aiframework && make install
git clone --depth 1 https://github.com/garrytan/gstack.git ~/.claude/skills/gstack && cd ~/.claude/skills/gstack && ./setup --no-prefix

2. Bootstrap your project

Scans your repo with 13 scanners and generates all configs. Deterministic — same repo, same output.

aiframework run --target ~/your-project

3. Open Claude Code

One command enhances your CLAUDE.md with stack-specific research. After this, just code.

cd ~/your-project && claude
/aif-ready
