// how it works

Four layers, one install

Everything installs together. Each layer handles a different phase of the AI development lifecycle. Together they create a feedback loop that gets smarter every session.

// scanning

Analysis Engine

13 scanners detect language, framework, commands, CI/CD, environment, domains, code symbols, and project archetype. Deterministic output: same repo, same result.

// generation

Config Generator

Produces CLAUDE.md, .claude/rules/, .claude/skills/, git hooks, CI workflows, AGENTS.md, .cursorrules, and a 31-file vault knowledge base.

// runtime

Session Intelligence

Watches your coding sessions. Extracts failure patterns. Bridges learnings between JSONL and markdown. Feeds insights back into your configs automatically.

// memory

Persistent Vault

Decisions, wiki pages, retro insights, and operational context survive across every session restart. Claude reads vault/memory/status.md at session start.

// automation

What runs without you doing anything

After the initial setup, the framework maintains itself. Health checks, type checking, linting, quality gates, config refresh, and session retros all fire via hooks.

01

Session Opens

automatic

SessionStart hook fires. Runs health check via sheal or aiframework verify. Project status surfaces immediately. Zero commands needed.

02

You Edit Code

automatic

PostToolUse hook fires on every file edit. Type checking runs for TypeScript, Rust, Go, and Python projects. Errors surface in real-time.

03

You Commit and Push

automatic

pre-commit runs lint. pre-push runs the full quality gate (lint + test + build), scans for secrets, detects config drift, and auto-refreshes if needed.

04

Session Closes

automatic

Stop hook fires. Runs session retrospective to extract learnings and failure patterns. Bridges insights to the JSONL store and vault memory.

05

Weekly Evolution

manual // safety gate

/aif-evolve reads all accumulated learnings and proposes rule improvements. Requires your approval before changing CLAUDE.md. This is intentional.

// stack

What gets installed

One make install sets up everything. sheal installs automatically if Node.js is available. Vault is generated by the bootstrap command. gstack adds 37 additional skills.

ComponentWhat it doesHow it installs
aiframework Core engine. 13 scanners, config generation, verification, learning bridge. make install automatic
sheal Runtime session intelligence. Health checks, retros, drift detection. Included in make install automatic
Vault Persistent memory. Decisions, wiki, retro insights across sessions. Generated by aiframework run automatic
gstack 37 skills for shipping, QA, browser testing, design, planning, security. git clone + ./setup automatic

requires: bash 3.2+ / jq 1.6+ / git 2.0+ / python 3.10+ (recommended) / node 18+ (for sheal)

// install

Three commands, two minutes

From zero to a fully AI-ready repo. Everything after this is automatic.

01 Install the framework

Installs aiframework + sheal. Then installs gstack (37 additional skills for Claude Code).

git clone https://github.com/evergonlabs/aiframework.git && cd aiframework && make install
git clone --depth 1 https://github.com/garrytan/gstack.git ~/.claude/skills/gstack && cd ~/.claude/skills/gstack && ./setup --no-prefix

02 Bootstrap your project

Scans your repo with 13 scanners. Generates CLAUDE.md, rules, skills, hooks, CI, vault, and docs.

aiframework run --target ~/your-project

03 Start coding

One command researches your stack and enhances everything. After this, just code. Hooks handle the rest.

cd ~/your-project && claude
