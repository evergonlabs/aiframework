Knowledge Vault

Three layers. Persistent memory.

A git-based knowledge graph using markdown + YAML frontmatter + wikilinks. Claude remembers across sessions.

raw/ ➡ wiki/ ➡ memory/ (strictly unidirectional)

Raw Layer

vault/raw/

Immutable source documents. Human-owned. Drop files here and the agent creates wiki summaries. Never modified by agents.

Wiki Layer

vault/wiki/

Processed knowledge — concepts, entities, comparisons. Auto-populated from the code index: module pages, architecture, API references.

Memory Layer

vault/memory/

Operational state — decisions, session notes, status tracking. Variable lifetime. The agent's working memory across sessions.

Get Started

Three steps. Two minutes.

No signup, no API key needed for the core pipeline. Everything is free and deterministic.

1

Clone aiframework

Get the tool on your machine.

git clone https://github.com/evergonlabs/aiframework.git
2

Run it on your repo

Point it at any project. It scans, indexes, generates, and verifies — all in one command.

./bin/aiframework run --target /path/to/your-project
3

Open Claude Code

Claude reads the generated CLAUDE.md and vault automatically. It now knows your entire project — structure, commands, invariants, module dependencies.

cd /path/to/your-project && claude
